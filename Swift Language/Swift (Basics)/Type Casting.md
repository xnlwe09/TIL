# Type Casting

## Type Casting(타입 캐스팅)이란?

**인스턴스의 타입을 확인하거나, 인스턴스의 타입을 슈퍼클래스 또는 서브클래스 타입처럼 다루기 위해 사용**

is 와 as 라는 연산자로 구현할 수 있으며, 이 두 연산자는 값의 타입을 확인하거나

다른 타입으로 변환하는 간단하고 표현적인 방법을 제공

## 사용 키워드

<aside>

### 📍`is`

인스턴스가 특정 타입인지 확인할 때 사용

```swift
class Animal {}
class Dog: Animal {}
class Cat: Animal {}

let dog = Dog()
print(dog is Dog)     // true
print(dog is Animal)  // true (상속받았으니까)
print(dog is Cat)     // false
```

</aside>

<aside>

### 📍`as`

업캐스팅(Upcasting) 또는 특정 타입으로 변환할 때 사용

(조건부 다운캐스팅 실패하면 nil)

</aside>

<aside>

## 🧷 업캐스팅(Upcasting)

하위 클래스 타입을 상위 클래스 타입으로 변환하는 것

항상 안전함 (`as`만 쓰면 됨)

→ 하위 클래스는 상위 클래스의 모든 속성과 기능을 포함하기 때문에 상위 클래스처럼 다룰 수 있음

(다형성을 활용할 수 있다는 장점!)

## 🧷 다운캐스팅(Downcasting)

상위 클래스 타입을 하위 클래스 타입으로 변환하는 것

항상 안전하진 않음 → 실제 인스턴스와 타입이 맞아야 성공함

실패할 수 있기 때문에 `as?`(조건부), `as!`(강제) 두 방식이 존재

(타입이 다른데 강제 캐스팅 하면 런타임 에러남)

- **업캐스팅** : 안전하지만 확장된 기능은 못 씀
- **다운캐스팅** : 위험할 수 있지만 하위 클래스 고유 기능 사용 가능
</aside>
