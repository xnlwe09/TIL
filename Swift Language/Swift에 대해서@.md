# Swift에 대해서@

## Swift란?

Apple이 2014년에 발표한 iOS, macOS 등의 개발을 위한 프로그래밍 언어

## Swift의 언어적 특성

Apple이 발표한 스위프트 언어적 특성은 **Safe(안전성)**, **Fast(신속성)**, **Expressive(더 나은 표현성)**으로 총 3가지이다.

<aside>

### ⛑️ Safe(안전성)

기본적으로 Swift는 안전한 프로그래밍을 지향함

Optional, guard, 타입 통제 등을 통해 프로그래머의 오류를 방지

</aside>

<aside>

### 💨 Fast(신속성)

C 언어와 비슷한 빠른 성능을 보이며, 실행속도 최적화 뿐만 아니라

지속적인 개량을 통해 컴파일러의 빠른 성능을 구현

</aside>

<aside>

### 🗯️ Expressive(표현성)

다양한 프로그래밍 언어의 장점들을 참고하여 보다 편하고 간결한, 가독성이 좋은 구조를 취함

</aside>

## Swift 프로그래밍 패러다임

**multiparadigm programming launguage**

Swift는 여러 가지 프로그래밍 언어의 특징과 패러다임을 차용한 **다중 패러다임 언어**이다.

(명령형과 객체지향 기반으로 한 함수형, 프로토콜 프로그래밍 패러다임을 지향)

*프로그래밍 패러다임 - 프로그래밍 언어가 추구하는 프로그래밍 수행기법 및 개발방식

<aside>

### 📍명령형 프로그래밍

어떤 문제를 어떻게(how) 수행할지의 관점에서 수행해야될 명령들을

순차적으로 적어 프로그래밍 하는 방식

</aside>

<aside>

### 📍객체지향 프로그래밍

Objective-C와의 연관성이 있어 기본적으로 객체지향의 특징을 취함

객체 단위의 소스코드 구성으로, 대규모 어플리케이션 개발 및 유지보수에도 용이하도록 설계되어 있음

</aside>

<aside>

### 📍함수형 프로그래밍

대규모 병렬처리를 쉽게 한다는 특징이 있고, 상태변화 보다는 함수 자체의 응용을 중요하게 여김

(Apple에서 특히나 강조하는 패러다임이라고 한다.)

가장 큰 특징은 함수를 일급 객체(First-class Citizen)으로 다룬다는 것이다.

(함수를 호출, 전달, 반환하는 동작만으로도 프로그램 구현이 가능하다는 의미)

→ 이같이 함수형 프로그래밍 패러다임으로 프로그램을 작성하면 상태 변화에 따른 부작용에서

자유로워진다는 특징이 있기 때문에 순수하게 기능 구현에 초점을 맞춰 설계가 가능

</aside>

<aside>

### 📍프로토콜 지향 프로그래밍

기존 클래스에서 구현할 수 있었던 캡슐화, 추상화, 접근 제어 등의 기능들을

구조체와 열거형에서 구현할 수 있음

참조 타입인 클래스의 인스턴스보다 값 타입을 사용함으로서 참조 타입의 참조 문제에서 좀 더

자유로워질 수 있고, 더 나은 효율성과 오류를 최소화할 수 있음

다중상속이 불가능한 한계를 뛰어넘을 수 있으며, 더 나은 추상화 메커니즘의 구현이 가능해짐

</aside>
